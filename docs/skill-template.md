## 常見藉口與反駁

Agent 傾向於以下理由跳過步驟，但這些理由都不成立：

| 藉口 | 反駁 |
|------|------|
| 「這個案例太簡單，不需要 [步驟 X]。」 | 簡單的案例也一樣適用流程，複雜度不是跳過的理由。 |
| 「我之後再補 [驗證/測試]。」 | 沒有驗證的完成等於沒有完成。現在就做。 |
| 「我已經知道答案了，可以跳過搜尋。」 | 訓練資料可能過時；文件確認只需幾秒鐘。 |
| 「使用者沒有明確要求 [步驟 Y]。」 | 流程是預設行為，不需要使用者每次都指定。 |

## 警示紅旗

出現以下跡象時，應立即暫停並重新評估：

- 輸出結果與預期差異過大
- 使用者表達困惑或質疑
- 遇到未在流程中涵蓋的邊界情況
- 步驟之間出現依賴衝突
