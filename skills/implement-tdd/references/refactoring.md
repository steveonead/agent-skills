# 重構候選

TDD 循環走完、所有測試都綠之後，找：

- **重複** → 抽成函式 / 類別
- **過長的方法** → 拆成 private helper（測試仍停在 public interface）
- **淺模組** → 合併或把實作加深
- **Feature envy**（方法動的資料不在自己類別） → 把邏輯搬到資料所在的地方
- **Primitive obsession**（到處傳基本型別） → 引入 value object
- **新程式揭露的既有程式問題** → 一併處理

---

## 重構三原則

1. **必須 GREEN**：紅燈狀態下不重構。紅燈重構會讓失敗原因變模糊——是新 behavior 沒實作，還是被你重構弄壞？
2. **每步跑測試**：測試是重構的安全網，沒跑等於沒網。重構步幅應該小到「壞了馬上看得出來是哪一步壞的」。
3. **不改 behavior**：重構的定義就是「改結構、不改 behavior」。發現要改 behavior，停下來另外起一輪 RED-GREEN。
