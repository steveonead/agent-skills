# Agent Skill 驗證檢查表

在 skill 部署前使用本檢查表做最終稽核。每一項都必須標記為「通過」，以確保 skill 易被發現、精簡且具確定性。

## 1. Metadata 與可發現性
*   [ ] **命名：** `name` 欄位為 1-64 字元，僅含小寫英文字母、數字或單一連字號。
*   [ ] **目錄一致：** `name` 欄位與父目錄名稱完全相同。
*   [ ] **Description 長度：** description 若含任一漢字（zh-TW／zh-CN／日文漢字）須少於 **200 Unicode 字元**；否則少於 **1,024** 字元。
*   [ ] **觸發條件最佳化：** description 同時包含使用情境（「Use when...」）與負面觸發條件（「Don't use for...」）。
*   [ ] **第三人稱語氣：** description 不出現 "I"、"me"、"my"、"you" 或 "your"。

## 2. 檔案結構與路徑
*   [ ] **扁平階層：** `scripts/`、`references/`、`assets/` 中所有檔案皆僅位於一層深處（無巢狀子資料夾）。
*   [ ] **標準資料夾：** 僅使用 `scripts/`、`references/` 與 `assets/`。
*   [ ] **不含人類文件：** 目錄中沒有 `README.md`、`CHANGELOG.md` 或 `INSTALLATION_GUIDE.md`。
*   [ ] **正斜線：** `SKILL.md` 中所有檔案路徑無論作業系統為何皆使用正斜線（`/`）。

## 3. 邏輯與指示（SKILL.md）
*   [ ] **精簡 context：** `SKILL.md` 檔案少於 500 行。
*   [ ] **祈使語氣：** 指示採用直接命令（例：「擷取」、「執行」、「驗證」）。
*   [ ] **確定性步驟：** 工作流程為編號、按時序排列的序列，並具清楚的決策樹。
*   [ ] **Progressive disclosure：** 大型 schema、模板或規則集置於 `references/` 或 `assets/`，僅在需要時讀取。
*   [ ] **用語具體：** 一致使用領域專屬術語（例：以「元件」取代「檔案」）。

## 4. Script 與確定性
*   [ ] **CLI 設計：** `scripts/` 中的腳本設計為可接受參數的小型 CLI。
*   [ ] **回饋迴路：** 腳本於成功時以具描述性的 `stdout` 回報、失敗時以 `stderr` 回報，以利 agent 自我修正。
*   [ ] **不含函式庫程式碼：** 腳本為單一用途；複雜邏輯交由 repository 的標準 CLI 或外部工具處理。

## 5. 錯誤處理
*   [ ] **邊界情境：** `SKILL.md` 包含「錯誤處理」章節，處理常見失敗狀態或遺漏設定。
*   [ ] **驗證：** `SKILL.md` 於適用處包含執行驗證腳本的步驟。
